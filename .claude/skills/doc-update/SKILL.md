# 문서 업데이트 스킬

현재 코드 구현 상태를 확인하고, 세 개의 핵심 문서를 동기화합니다.

## 문서 역할 (중요)

| 문서 | 역할 | 우선순위 |
|------|------|----------|
| **PRD.md** | 기준 문서 (Master) - 전체 요구사항 정의 | 1순위 |
| **PROGRESS.md** | 세부 진행 현황 - PRD 항목을 세분화하여 추적 | 2순위 |
| **Roadmap.md** | UI/UX 와이어프레임 - 화면 설계 및 구현 상태 | 3순위 |

**핵심 원칙:**
- PRD.md가 기준 → PROGRESS.md와 Roadmap.md가 따라감
- PROGRESS.md는 PRD 항목을 더 세부적으로 나눠서 추적
- Roadmap.md는 UI 화면 단위로 구현 완료 표시

## 대상 문서

@docs/01-planning/PRD.md
@docs/05-tracking/PROGRESS.md
@docs/01-planning/Roadmap.md

## 업데이트 절차

### 1단계: 현재 구현 상태 확인

다음을 확인합니다:
- `src/app/` 폴더의 페이지 구조
- `src/app/api/` 폴더의 API 라우트
- `src/components/` 폴더의 컴포넌트
- `src/types/` 폴더의 타입 정의

### 2단계: PRD.md 기준 비교

PRD.md의 체크박스 항목을 기준으로:
- 구현된 기능 중 체크 안 된 항목 찾기
- 누락된 기능 찾기

### 3단계: 문서 업데이트 (순서 중요)

**1. PRD.md 먼저 업데이트 (기준 문서):**
- 기능 체크박스 업데이트 (`[ ]` → `[x] ✅`)
- Phase 섹션 진행 상황 업데이트
- `updated` 타임스탬프 갱신

**2. PROGRESS.md 업데이트 (세부 항목):**
- PRD 항목에 대응하는 세부 작업 체크
- 작업 완료 표시 (`[ ]` → `[x] ✅ (날짜)`)
- 섹션 제목에 완료 표시 (✅ 완료)
- `updated` 타임스탬프 갱신

**3. Roadmap.md 업데이트 (UI 화면):**
- 와이어프레임 섹션에 구현 완료 표시 (✅ 구현 완료)
- 실제 UI와 다른 와이어프레임 수정
- Phase 마일스톤 체크박스 업데이트
- `updated` 타임스탬프 갱신

## 동기화 규칙 (중요)

### 1. 체크박스 형식
```markdown
# PRD.md
- [ ] 미완료 기능
- [x] 완료된 기능 ✅

# PROGRESS.md
- [ ] 미완료 기능
- [x] 완료된 기능 ✅ (2026-02-06)

# Roadmap.md
### 2.x 섹션명 ✅ 구현 완료 (2026-02-06)
```

### 2. 타임스탬프 형식
```yaml
updated: 2026-02-06T16:00
```

### 3. 상태 계산 로직 (개별 일정표)
| 상태 | 조건 |
|------|------|
| 예정 | 계획 시작일 > 오늘 AND 실적 시작일 없음 |
| 진행중 | 실적 시작일 있음 AND 실적 종료일 없음 |
| 완료 | 실적 종료일 있음 |
| 지연 | 계획 시작일 <= 오늘 AND 실적 시작일 없음 |

### 4. 색상 규칙 (간트차트/상태 배지)
| 상태 | 색상 |
|------|------|
| 예정 | 회색 |
| 진행중 | 주황색 |
| 완료 | 초록색 |
| 지연 | 빨간색 |
| 계획 | 파란색 |

## 주의사항

1. **PRD가 기준**: PRD.md를 먼저 업데이트하고 나머지 문서가 따라감
2. **PROGRESS는 세분화**: PRD의 큰 항목을 작은 단위로 나눠서 추적
3. **Roadmap은 UI 중심**: 화면별 와이어프레임 + 구현 완료 표시
4. **완료 항목 관리**:
   - 완료 체크(`[x]`)는 유지
   - 개선/수정 시: 날짜만 업데이트 (예: `✅ (2026-02-05)` → `✅ (2026-02-06 개선)`)
   - 기능 변경 시: 세부 내용 추가/수정
5. **날짜 정확성**: 완료일은 실제 구현된 날짜로 기록
6. **와이어프레임 일치**: Roadmap.md의 와이어프레임이 실제 UI와 다르면 수정

└── 2.6 프로젝트 상세 화면 ✅ 구현 완료
