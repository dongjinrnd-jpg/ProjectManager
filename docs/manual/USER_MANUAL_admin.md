## 1. 시스템 개요

### 1.1 접근 가능 메뉴

관리자는 모든 메뉴에 접근할 수 있습니다:

| 메뉴 | 아이콘 | 설명 | 권한 |
|------|--------|------|------|
| 대시보드 | 📊 | 팀 전체 프로젝트 현황 | 조회 |
| 프로젝트 | 📁 | 프로젝트 생성/수정/삭제 | CRUD + 삭제 |
| 업무일지 | 📝 | 팀원 업무일지 조회 | CRUD |
| 고급 검색 | 🔍 | 업무일지 통합 검색 | 조회 |
| 전체 일정 | 📅 | 캘린더 형태의 일정 | 조회 |
| 주간보고 | 📋 | 주간 단위 업무 보고서 | CRUD |
| 경영진 | 👔 | 경영진 대시보드 | 조회 |
| 사용자 관리 | 👥 | 사용자 계정 관리 | CRUD |
| 시스템 설정 | ⚙️ | 공통 코드 관리 | 설정 |

## 2. 기본 기능

엔지니어 메뉴얼의 모든 기능을 사용할 수 있습니다:
- 대시보드

- 업무일지 작성

### 4.3 입력 항목 설명

| 항목 | 필수 | 설명 | 예시 |
|------|------|------|------|
| 아이디 | ● | 로그인 ID | hong |
| 이름 | ● | 사용자 이름 | 홍길동 |
| 이메일 | - | 이메일 주소 | hong@company.com |
| 비밀번호 | ● | 초기 비밀번호 (8자 이상 권장) | |
| 권한 | ● | 사용자 역할 | engineer |
| 소속 | ● | 부서/팀 | R&D팀 |
| 활성화 | - | 계정 활성화 여부 | ✅ |

### 4.4 역할(Role) 설명

| 역할 | 코드 | 권한 범위 |
|------|------|-----------|
| 일반 사용자 | user | 조회만 가능 |
| 엔지니어 | engineer | 프로젝트/업무일지 CRUD |
| 관리자 | admin | 사용자/시스템 관리 포함 |
| 경영진 | executive | 조회 + 경영진 대시보드 |
| 시스템 관리자 | sysadmin | 시스템 전체 관리 |

### 4.5 비밀번호 재설정

1. 사용자 목록에서 해당 사용자의 **[⚙️]** 클릭
2. **[비밀번호 재설정]** 버튼 클릭
3. 새 비밀번호 입력 후 **[저장]**

> **보안 주의**: 비밀번호 재설정 후 사용자에게 초기 비밀번호를 안전하게 전달하세요.

## 6. 관리자 권한 주의사항

### 6.1 프로젝트 삭제

> **경고**: 프로젝트 삭제 시 관련 데이터도 함께 삭제됩니다.

삭제되는 데이터:
- 프로젝트 기본 정보
- 관련 업무일지
- 세부추진항목
- 주간보고
- 회의록

**복구 불가능하므로 삭제 전 반드시 확인하세요.**

### 6.2 사용자 관리 주의사항

| 상황 | 주의사항 |
|------|----------|
| 역할 변경 | 즉시 권한이 적용됨 |
| 계정 비활성화 | 해당 사용자 로그인 불가 |
| 계정 삭제 | 작성한 데이터는 유지되나 작성자 정보 연결 해제 |
| 본인 계정 | 본인 역할 변경 불가 |

### 6.3 퇴사자 처리

1. 해당 사용자의 계정을 **비활성화** 처리
2. 필요시 담당 프로젝트의 팀장/팀원 재지정
3. 계정 삭제는 신중히 결정 (데이터 연결 해제됨)

## 7. 자주 묻는 질문

### Q1. 팀원의 업무일지를 수정할 수 있나요?

> 아니요. 업무일지는 작성자 본인만 수정 가능합니다.
> 내용 수정이 필요하면 해당 팀원에게 요청해주세요.

### Q2. 삭제된 프로젝트를 복구할 수 있나요?

> 아니요. 삭제된 데이터는 복구할 수 없습니다.
> 삭제 전 신중히 확인해주세요.

### Q3. 경영진 역할로 변경하면 어떻게 되나요?

> 경영진은 조회 중심 역할입니다.
> 프로젝트 생성/수정 권한이 없어질 수 있으니 주의하세요.

### Q4. 사용자 비밀번호를 확인할 수 있나요?

> 아니요. 비밀번호는 암호화되어 저장되므로 확인할 수 없습니다.
> 비밀번호 분실 시 재설정만 가능합니다.

### Q5. 공통 코드를 삭제하면 기존 데이터는 어떻게 되나요?

> 기존 데이터에는 영향이 없습니다.
> 단, 새로운 데이터 입력 시 해당 옵션을 사용할 수 없게 됩니다.
