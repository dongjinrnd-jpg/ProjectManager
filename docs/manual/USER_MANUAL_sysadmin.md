## 1. 시스템 개요

### 1.1 시스템 관리자 역할

시스템 관리자는 시스템 전반의 설정과 사용자 관리를 담당합니다.

> **참고**: 업무일지/주간보고 메뉴는 sysadmin에게 제공되지 않습니다.
> 업무 기록이 필요한 경우 engineer 또는 admin 역할의 별도 계정을 사용하세요.

### 1.2 접근 가능 메뉴

| 메뉴 | 아이콘 | 설명 | 권한 |
|------|--------|------|------|
| 대시보드 | 📊 | 전체 프로젝트 현황 요약 | 조회 |
| 프로젝트 | 📁 | 프로젝트 목록 조회/관리 | CRUD |
| 전체 일정 | 📅 | 캘린더 형태의 일정 조회 | 조회 |
| 경영진 | 👔 | 경영진 대시보드 | 조회 |
| 사용자 관리 | 👥 | 사용자 계정 관리 | CRUD |
| 시스템 설정 | ⚙️ | 공통 코드, 시스템 설정 | 설정 |

### 2.2 사용자 등록

1. **[+ 사용자 추가]** 클릭
2. 정보 입력:

```
┌─────────────────────────────────────────────────────────────────┐
│  👤 사용자 등록                                    [X 닫기]     │
│  ─────────────────────────────────────────────────────────────  │
│                                                                  │
│  이름 *         [홍길동                    ]                    │
│                                                                  │
│  이메일 *       [hong@company.com          ]                    │
│                  ※ 로그인 ID로 사용됩니다                       │
│                                                                  │
│  비밀번호 *     [●●●●●●●●●●                ]                    │
│                  ※ 8자 이상 권장                                │
│                                                                  │
│  역할 *         [engineer         ▼]                            │
│                 ┌──────────────────┐                            │
│                 │ user      - 일반 사용자 (조회만)              │
│                 │ engineer  - 엔지니어 (CRUD) ✓                 │
│                 │ admin     - 관리자 (사용자/시스템 관리)       │
│                 │ executive - 경영진 (조회 + 대시보드)          │
│                 │ sysadmin  - 시스템 관리자                     │
│                 └──────────────────┘                            │
│                                                                  │
│  소속 *         [R&D팀             ▼]                           │
│                                                                  │
│          [취소]            [저장]                                │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 입력 항목 상세

| 항목 | 필수 | 설명 | 예시 |
|------|------|------|------|
| 이름 | ● | 사용자 이름 | 홍길동 |
| 이메일 | ● | 로그인 ID (이메일 형식) | hong@company.com |
| 비밀번호 | ● | 초기 비밀번호 (8자 이상 권장) | |
| 역할 | ● | 권한 등급 | engineer |
| 소속 | ● | 부서/팀명 | R&D팀 |

### 2.4 역할(Role) 상세

| 역할 | 코드 | 권한 범위 | 주요 용도 |
|------|------|-----------|-----------|
| 일반 사용자 | user | 조회만 가능 | 외부 열람자, 인턴 |
| 엔지니어 | engineer | 프로젝트/업무일지 CRUD | 프로젝트 담당자 |
| 관리자 | admin | 사용자/시스템 관리 포함 | 팀장, 부서장 |
| 경영진 | executive | 조회 + 경영진 대시보드 | 임원, 경영진 |
| 시스템 관리자 | sysadmin | 시스템 전체 관리 | IT 담당자 |

### 2.5 비밀번호 재설정

1. 사용자 목록에서 **[수정]** 클릭
2. **[비밀번호 재설정]** 버튼 클릭
3. 새 비밀번호 입력
4. **[저장]** 클릭

> **보안 주의**: 비밀번호 재설정 후 사용자에게 초기 비밀번호를 안전하게 전달하세요.
> 이메일, 메신저 등을 통해 직접 전달하고, 첫 로그인 시 변경을 권장하세요.

### 2.6 사용자 삭제

1. 사용자 목록에서 **[삭제]** 클릭
2. 확인 팝업에서 **[삭제]** 클릭

> **주의**:
> - 삭제된 사용자는 복구할 수 없습니다
> - 해당 사용자의 업무일지는 유지되지만 "작성자 없음"으로 표시됩니다
> - 퇴사자는 삭제보다 **비활성화** 처리를 권장합니다

## 5. 시스템 운영 가이드

### 5.1 정기 점검 사항

| 주기 | 점검 항목 | 조치 |
|------|-----------|------|
| **일간** | 로그인 현황 확인 | 비정상 접근 시 계정 확인 |
| **주간** | 비활성 사용자 확인 | 30일 이상 미접속 계정 확인 |
| **월간** | 불필요한 데이터 정리 | 테스트 데이터 삭제 |
| **분기** | 권한 검토 | 퇴사자/부서이동자 권한 정리 |

### 5.2 데이터 백업

Google Sheets 기반이므로:
- Google Sheets 자체 버전 기록 활용 (파일 > 버전 기록)
- 정기적으로 스프레드시트 다운로드 권장 (xlsx, csv)

### 5.3 보안 관리

| 항목 | 권장 사항 |
|------|-----------|
| **비밀번호 정책** | 8자 이상, 영문+숫자 조합 |
| **계정 관리** | 퇴사자 계정 즉시 비활성화/삭제 |
| **권한 관리** | 최소 권한 원칙 적용 |
| **접근 로그** | 정기적인 로그 확인 |

### 5.4 퇴사자 처리 절차

1. 해당 사용자 계정 **비활성화** 처리
2. 담당 프로젝트의 팀장/팀원 재지정
3. 필요시 인수인계 확인 후 계정 삭제

## 6. Google Sheets 구조

### 6.1 시트 목록

| 시트명 | 용도 | 주요 컬럼 |
|--------|------|-----------|
| **Users** | 사용자 정보 | id, name, email, password, role, division |
| **Projects** | 프로젝트 정보 | id, customer, item, leaderId, startDate, endDate |
| **WorkLogs** | 업무일지 | id, projectId, date, content, issue |
| **WeeklyReports** | 주간보고 | id, weekNumber, division, content |
| **MeetingMinutes** | 회의록 | id, projectId, title, meetingDate |
| **Favorites** | 즐겨찾기 | id, userId, projectId |
| **SavedSearches** | 저장된 검색 | id, userId, name, filtersJson |
| **ScheduleItems** | 세부추진항목 | id, projectId, stage, itemName |

### 6.2 직접 수정 시 주의사항

> **경고**: Google Sheets 직접 수정은 권장하지 않습니다.

불가피한 경우 주의사항:
- **ID 컬럼**은 절대 수정하지 마세요 (참조 무결성 깨짐)
- **날짜 형식** 유지 (YYYY-MM-DD)
- **빈 행 삽입** 금지
- 수정 전 해당 시트 **백업** 필수

## 7. 트러블슈팅

### 7.1 로그인 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 비밀번호 분실 | - | 사용자 관리에서 비밀번호 재설정 |
| 권한 오류 | 역할 설정 오류 | 역할(Role) 확인 및 수정 |
| 로그인 불가 | 계정 비활성화 | 계정 상태 확인 |

### 7.2 데이터 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 데이터 누락 | 동기화 지연 | 페이지 새로고침, Google Sheets 직접 확인 |
| 중복 데이터 | 동시 저장 | 중복 데이터 수동 삭제 |
| 연결 오류 | API 문제 | 잠시 후 재시도, API 상태 확인 |

### 7.3 시스템 오류

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 500 에러 | Google API 할당량 초과 | 24시간 대기 또는 할당량 상향 |
| 느린 응답 | 데이터량 과다 | 불필요한 데이터 정리 |
| 배포 실패 | 빌드 에러 | Vercel 로그 확인, 코드 수정 |

### 7.4 Google API 할당량

- 기본 할당량: 분당 60회 요청
- 초과 시 24시간 후 자동 리셋
- 필요 시 Google Cloud Console에서 할당량 상향 요청

## 8. 자주 묻는 질문

### Q1. 왜 업무일지 메뉴가 없나요?

> sysadmin은 시스템 관리 역할입니다.
> 업무 기록이 필요하면 별도의 engineer 계정을 사용하세요.

### Q2. 삭제한 사용자를 복구할 수 있나요?

> 아니요. 동일한 정보로 새로 등록해야 합니다.
> 단, 기존 업무일지와의 연결은 복구되지 않습니다.
> **권장**: 삭제보다 비활성화 처리

### Q3. Google Sheets 할당량 초과 시 어떻게 하나요?

> 1. 24시간 대기 (자동 리셋)
> 2. Google Cloud Console에서 할당량 상향 요청
> 3. 불필요한 API 호출 최소화 검토

### Q4. 새로운 개발 단계를 추가하고 싶습니다.

> 시스템 설정 > 개발 단계 관리에서 **[+ 커스텀 단계 추가]** 클릭
> 추가한 단계는 프로젝트 생성 시 선택 가능

### Q5. 공통 코드를 삭제하면 기존 데이터는 어떻게 되나요?

> 기존 데이터에는 영향이 없습니다.
> 단, 새로운 데이터 입력 시 해당 옵션을 사용할 수 없게 됩니다.
